What's makes Tahoma2D different?

By Jeremy Bullock | August 15, 2020


Coming from OpenToonz?

If you are coming to Tahoma2D from OpenToonz, it will feel very familiar. But you will start to notice small changes here and there that help make the experience better.

The list of changes is pretty big. I’ll go into more depth about some of these in the future.

UI

  • Updated styles by konero
  • New status bar at the bottom of the program to give hints and tips. (This can be hidden.)
  • Removed Auto and Apply buttons from style editor
  • Big function editor overhaul with channel colors
  • Simplified default view in the viewer - need to do this to the combo viewer too
  • Big new style button in the palette viewer.
  • Reorganized buttons in the palette viewer
  • Updated how the style editor shows colors and swatches.
  • Simplified camera settings window.
  • Simplified output settings window.
  • Renamed default style to medium
  • Dark is the default style.
  • Fx levels now show an icon in the timeline and xsheet.
  • Removed toggle orientation button in the xsheet - it is now in the right click menu.
  • Revamped rooms
  • The memo buttons have been moved lower and the old new memo button now is a create level button.
  • Time indicator in the viewer
  • Panel titlebars are now hidden by default when the rooms are locked
  • Table hidden by default

Stop Motion

  • Huge overhaul of usability and refinement
  • Camera Test Shots panel in Stop Motion
  • Live View Exposure Offset for Stop Motion
  • Stop Motion can now show the scene on monitors during capture for lighting or backgrounds.
  • Raw image support for Canon cameras.
  • Allow capturing 2’s, 3’s. . .
  • Much more. . .

Drawing

  • Toonz Raster levels are now Smart Raster Levels
  • New perspective assistants on raster and Smart Raster levels.
  • Holding shift on raster and Smart Raster levels creates straight lines
  • Holding control on raster and Smart Raster levels locks to vertical, horizontal or diagonal lines.
  • New default palettes. Set up and save palettes for quick reuse.

Usability

  • Projects can now be saved anywhere in a much more simplified way.
  • Render is now Save and Render - Scenes are saved before rendering to prevent lost work.
  • The lock in the top right now locks rooms and panes at the same time. No more lock options in two places.
  • Pixels are the only supported unit - no inches or mm
  • Simplified list of supported export types
  • Double clicking in graph window fits the contents to the window
  • Changed E/W and N/S to X and Y
  • Removed customizable menubars
  • Autosave no longer shows a pop up
  • Changed clicking on a folder in file browser to need double clicking to open the folder
  • Added an indicator if the frame rate of the viewer is not the same as the scene
  • New “Delete Frame” command in the level strip.
  • New “Paste as a Copy” command in the timeline/xsheet.
  • You can now change the parent of a column from the xsheet. No need to switch to the schematic view.
  • Hex numbers for colors can now be used in the Style Editor
  • You can now search for commands when customizing the xsheet toolbar and command bar.

Bug Fixes

  • Fixed a bug with room names that had a space.
  • All fixes current with OT.
  • Fix pan, rotate and zoom to work with shortcuts without losing selections
  • Fix error where brush wouldn’t work correctly on raster levels of different sizes
  • Fixed the playback fps color not resetting after pause
  • Fixed some strange panel docking issues.
  • Many more. . .
